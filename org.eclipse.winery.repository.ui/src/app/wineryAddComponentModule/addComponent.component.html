The component version specifies the components' external version defined by the creator of the software. For example, Apache Tomcat in version 8.5.1 has a component version 8.5.1. Winery adds management to the software which is versioned independently of the softwares' version. The version inside Winery is called management version and is mandatory. It consists of a winery version and a work in progress (wip) version. The winery version identifies the internal version. When working on the TOSCA definition, a wip version is appended until the TOSCA definition is stable and released in the current winery version. Upon the creation of a new component, both management versions are set automatically with their initial values of 1. The generated name is displayed in the 'Final name' field. For example, the final id for Tomcat in version 8.5.1 is: Tomcat_8.5.1-w1-wip1.
